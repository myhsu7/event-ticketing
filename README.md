# exYahoo Summer Party QR-Code 報名與簽到系統 - 設定指南

本專案提供一個基於 **Google Apps Script (GAS)** 的免伺服器解決方案。您只需使用現有的 Google 表單與 Google 試算表，配合本專案的程式碼，即可自動化核對匯款、發送 QR Code 門票、以及現場手機掃描簽到。

---

## 📂 專案檔案說明
1. [Code.gs](file:///Users/myhsu/Devel/exYahoo/Code.gs): Google Apps Script 後端邏輯，負責處理資料庫編輯、UUID 產生、發送 Gmail，以及簽到 API。
2. [Index.html](file:///Users/myhsu/Devel/exYahoo/Index.html): 簽到成功/失敗的行動端網頁，採用精美的 exYahoo 紫色調與玻璃擬物化 (Glassmorphic) 設計。

## 🛠️ 第一步：準備 Google 試算表欄位
請確保您的 Google 表單所連結的試算表中，**首列 (第一列) 的標頭名稱** 必須精確包含以下欄位（順序不限，但字元需完全一致）：

1. **`姓名`** (或 `聯絡人姓名`，作為信件稱呼)
2. **`電子郵件地址`** (或 `電子郵件` / `Email`，作為收信郵箱)
3. **`對帳狀態`** (您可以在此欄設定資料驗證，建立下拉選單：`待核對`、`匯款完成`、`款項不符`)
4. **`Ticket UUID`** (留空，系統將在核對完成後自動填入)
5. **`入場狀態`** (留空，當天掃描後會寫入 `已入場`)
6. **`入場時間`** (留空，當天掃描後會寫入實際時間)
7. **`備註`** (或 `Remarks` / `備註欄`，非必填。若要進行測試發送，請在此欄填入 `test`)
8. **`門票發送狀態`** (或 `門票狀態` / `入場券發送狀態`，留空即可。門票發送成功後系統會自動寫入 `已發送`，以便過濾防止重複發送)

---

## 📝 第二步：貼入 Apps Script 程式碼
1. 在該 Google 試算表中，點選上方選單的 **「擴充功能」** -> **「Apps Script」**。
2. 清空預設的 `Code.gs` 內容，並將 [Code.gs](file:///Users/myhsu/Devel/exYahoo/Code.gs) 的程式碼全部複製貼上。
3. 點選左側「檔案」旁邊的 **「+ (新增檔案)」** -> 選擇 **「HTML」**。
4. 將檔案命名為 **`Index`** (注意：首字母大寫，擴充名為 `.html`，GAS 會自動加上，所以檔名輸入 `Index` 即可)。
5. 將 [Index.html](file:///Users/myhsu/Devel/exYahoo/Index.html) 的程式碼全部複製貼上。
6. 點選上方存檔按鈕（磁碟片圖示）。

## 🔒 第三步：設定安全金鑰
為了防止他人隨意猜出 URL 並自行標記已進場，系統設計了 Token 驗證機制。
1. 在 Apps Script 畫面中，點選左側的 **「專案設定」**（齒輪圖示）。
2. 捲動到最下方，找到 **「指令碼屬性 (Script Properties)」**，點選 **「新增指令碼屬性」**。
3. 新增屬性：
   * **屬性 (Property)**: `SECRET_TOKEN`
   * **值 (Value)**: `請輸入自訂的複雜密碼字串` (例如: `exyahooSummerParty2026SecureToken!`)
4. 點選 **「儲存指令碼屬性」**。

---

## ⚙️ 第四步：設定 Installable 編輯觸發器
由於 Apps Script 的簡易編輯觸發器 (`onEdit`) 沒有權限寄信與呼叫外部服務，我們必須設定「可安裝的觸發器 (Installable Trigger)」：
1. 點選 Apps Script 左側選單的 **「觸發器」**（時鐘圖示）。
2. 點選右下角 **「新增觸發器」**。
3. 進行以下設定：
   * **選取要執行的函式**: `handleEdit`
   * **選取要執行的部署作業**: `導向至最新版本` (Head)
   * **選取活動來源**: `來自試算表`
   * **選取活動類型**: `編輯時`
4. 點選 **「儲存」**。此時會彈出視窗要求授權存取您的 Google 帳號（用以執行試算表讀寫與 Gmail 寄信），請按照提示完成授權。

## 🚀 第五步：部署網頁應用程式 (Web App)
1. 在 Apps Script 畫面右上角，點選 **「部署」** -> **「新增部署」**。
2. 點選左上角選單（齒輪圖示），選擇 **「網頁應用程式 (Web App)」**。
3. 填寫設定：
   * **說明**: `exYahoo Summer Party Check-in v1`
   * **執行身分**: `我` (Me) - 代表使用您的帳號權限來寫入試算表
   * **誰能存取**: `任何人` (Anyone) - 讓當天現場的手機能夠訪問此網頁
4. 點選 **「部署」**。
5. 部署完成後，畫面上會出現 **「網頁應用程式 URL」**。**請將該 URL 完整複製下來**。
6. 回到 **「專案設定」**（齒輪圖示）的 **「指令碼屬性」**，新增另一個屬性：
   * **屬性 (Property)**: `WEB_APP_URL`
   * **值 (Value)**: `貼上剛剛複製的 Web App URL` (例如: `https://script.google.com/macros/s/xxxx/exec`)
7. 點選 **「儲存指令碼屬性」**。

## 📅 第五點五步：活動日曆設定說明 (新增功能)
本系統已整合 **Google Calendar** 與 **Yahoo Calendar**，會自動為來賓建立行事曆事件，並在信件中附帶一鍵加入按鈕。

### 1. 修改活動日曆資訊
在開始對帳發信前，請先開啟 [Code.gs](file:///Users/myhsu/Devel/exYahoo/Code.gs)，在最上方的 **「活動行事曆資訊設定」** 區域，修改您的活動詳細資訊：
* `EVENT_TITLE`: 活動名稱 (例如：`exYahoo Summer Party`)
* `EVENT_START`: 活動開始時間 (例如：`2026-07-25T18:00:00+08:00`)
* `EVENT_END`: 活動結束時間 (例如：`2026-07-25T21:30:00+08:00`)
* `EVENT_LOCATION`: 活動地點
* `EVENT_DESCRIPTION`: 活動說明文案

### 2. 運作模式
* **自動加為日曆受邀者**：當對帳完成時，系統會自動在您的 Google 日曆中建立一個主活動，並將來賓的 Email 新增為 Guest。
  * **Gmail 使用者** 會直接在 Google Calendar 上顯示此活動，並收到 Google 的日曆受邀通知。
  * **Yahoo 使用者** 會自動收到一封含 `.ics` 附件的邀請信，讓他們能在 Yahoo Mail 中直接點選「接受」並加到行事曆。
* **信件一鍵加入按鈕**：門票信件中已內建「+ Google 日曆」與「+ Yahoo 日曆」按鈕，方便來賓手動快速點選加入。

## 🧪 第六步：測試流程與「測試模式」

本系統提供兩種測試方式，以防測試信寄給真實來賓：

### 方式 1：手動執行測試（極力推薦）
當您想測試信件排版，但不想影響任何真實報名的來賓時：
1. 在試算表找出一列（或建立新的一列），填寫您自己的姓名與測試 Email。
2. 將該列的 **`對帳狀態`** 設定為 **`匯款完成`**。
3. 在 **`備註`** (或 `Remarks` / `備註欄`) 填入 **`test`**。
4. 在 Apps Script 編輯器上方，將執行函式下拉選單選為 **`runSendTicketsTest`**，點選 **「執行」**。
5. 此時，系統只會處理備註包含 `test` 且狀態為 `匯款完成` 的列。其它所有正式來賓的資料都會被安全跳過。
6. ⚠️ **提示**：在測試模式下，即使該測試列已經有 UUID，每次執行都會重新產生新 UUID 並重發信件，方便您重複測試信件與 QR Code。

### 方式 2：全域測試模式（自動觸發）
如果您想在直接編輯試算表時進行測試：
1. 打開 [Code.gs](file:///Users/myhsu/Devel/exYahoo/Code.gs)，在最上方將 `var GLOBAL_TEST_MODE = false;` 改為 **`true`** 並存檔。
2. 這時，不論是手動編輯還是批次執行，只要 **`備註`** 欄位沒有寫 **`test`**，系統就會自動跳過，不進行任何處理。
3. 測試完畢後，請記得將 `GLOBAL_TEST_MODE` 改回 `false`。

## 📱 活動當天：如何現場簽到與安全機制

### 1. 現場簽到流程
1. 主辦人員使用任何智慧型手機（iOS 內建相機 / Android 掃描器），直接掃描來賓出示的 QR Code 門票。
2. 掃描後，手機瀏覽器會打開簽到網頁：
   * **第一次掃描**：顯示 **綠色「簽到成功」** 畫面，並顯示來賓姓名。試算表中的 **`入場狀態`** 會自動變更為 `已入場`，並記錄 **`入場時間`**。
   * **重複掃描**：顯示 **黃色「重複簽到」** 警示，並會列出該門票的首次入場時間，防止他人一票多用。
   * **無效票券/非工作人員存取**：顯示 **紅色「簽到失敗」** 或 **灰色「無權限」** 畫面。

### 2. 🛡️ 安全機制：防範來賓提前掃描
為了防範來賓出於好奇，在活動開始前幾天就私自掃描 QR Code 導致系統提早寫入簽到記錄：
* **動態開啟時間**：系統會自動抓取 `EVENT_START` 時間，並於**活動開始前 1 小時**（例如 `7/24 17:00`）才正式開放簽到。
* **提早掃描限制**：在開放時間前，若非測試列的來賓掃描，系統會自動阻擋且**不寫入**試算表，網頁將呈現黃色的 **「尚未開放」** 警告，並提示正式開放時間。
* **工作人員測試例外**：若該來賓在試算表中的 **`備註`** 欄位包含 **`test`**，則會自動忽略時間限制，以利主辦單位於活動前幾天隨時測試掃描頁面。

## 🤖 兆豐自動對帳系統 (Gmail API + 試算表自動同步)

本系統新增了自動讀取信箱、比對匯款後五碼與金額、自動匯入並同步報名名單與對帳表的功能。

### 1. 運作流程
1. **信件讀取**：每日定時搜尋兆豐銀行發送的數位帳戶轉入通知信（未讀、金額 1000 元）。
2. **後五碼解析**：從信件的「交易備註」中擷取數字，並取最右側 5 碼作為匯款帳號後五碼。
3. **報名名單更新 (Case A/B)**：
   * **已存在於報名名單**：比對銀行交易筆數與名單筆數。若相符，將「對帳狀態 (N 欄)」改為 `匯款完成`。
   * **不存在於報名名單**：前往 `表單回覆 1` 比對後五碼。若筆數相符，則依表單「匯款時間」由早到晚順序，將資料自動 Append 到 `報名名單` 最下方，自動編號（如 `045`，文字格式）且設定為 `匯款完成`。
   * **筆數不符或找不到**：跳過不處理，列入異常清單。
4. **對帳表同步**：自動比對 `匯款對帳`（G 欄）與 `報名名單`（J 欄）的後五碼，進行一對一依序配對：
   * **配對成功**：在 `匯款對帳` 寫入報名序號（強制文字格式，不遺失前導零）、姓名、Email（優先取表單 O 欄，次取名單 D 欄）、對帳狀態寫入 `匯款完成`。系統會**嚴格保留原有的 Ticket UUID 與門票發送狀態**，不進行覆蓋。
   * **配對失敗**：狀態寫入 `找不到對應：<後五碼>` 或 `筆數不符：<後五碼>`。
5. **自動寄發門票**：對帳完成後，系統會**自動呼叫門票寄送流程**。若有新標記為 `匯款完成` 且尚未發送門票的來賓，會自動生成 UUID、寄發確認信與 QR Code，並發送 Google 日曆專屬邀請。
6. **異常回報**：執行過程中若有金額錯誤、筆數不對或找不到名單等異常，會在執行結束後自動發送彙整信件給您，並支援 Telegram Bot 即時推播。

### 2. 啟動與觸發器設定

為了讓系統每天自動運行，我們可以直接利用程式碼中內建的輔助工具完成設定：
1. **一鍵設定排程觸發器**：
   * 打開 Google Apps Script 編輯器，在上方執行函式下拉選單選取 **`setupAutoReconciliationTrigger`**。
   * 點選 **「執行」**。
   * **效果**：這會自動為您在背景註冊每日 **12:00**、**18:00**、**21:00** 執行的時間驅動觸發器，不需手動一個一個新增。
2. **手動即時對帳**：
   * 若想立刻執行對帳，可在選單選取 **`runAutoReconciliation`** 並點選 **「執行」**。
3. **異常通知信箱與 Telegram 設定**：
   * 您可以編輯 [Code.gs 最上方的設定區](file:///Users/myhsu/Devel/exYahoo/Code.gs#L572)，填入 `ADMIN_NOTIFY_EMAIL` 以指定警告信收件地址，或設定 `TELEGRAM_BOT_TOKEN` 與 `TELEGRAM_CHAT_ID` 以啟用 Telegram 自動警告推播。
